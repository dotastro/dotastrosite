# Contributing to the .Astronomy Archive

Thank you for helping build the .Astronomy archive! We welcome contributions from all .Astronomy alumni and the wider community.

## How to Contribute

1. Fork this repository
2. Create a branch for your changes
3. Make your edits following the formats below
4. Submit a pull request

## Adding Talk Information

In the relevant event folder, edit `talks.md` and add entries in this format:

```markdown
### Talk Title

**Speaker:** Name (Affiliation)
**Abstract:** Brief description of the talk
**Links:** [Slides](url) | [Video](url)
```

## Adding Hack Day Projects

In the relevant event folder, edit `hacks.md` and add entries in this format:

```markdown
### Hack Title

**Team:** Name1, Name2, Name3
**Description:** What the hack does/did
**Links:** [Code](github-url) | [Demo](url)
**Status:** Completed / In Progress / Concept
```

## Adding Unconference Sessions

In the relevant event folder, edit `unconference.md` and add entries in this format:

```markdown
### Session Title

**Proposed by:** Name
**Summary:** Brief description of what was discussed
**Key outcomes:** What came out of the discussion
```

## Adding Participants

In the relevant event folder, edit `participants.md` and add your name in alphabetical order:

```markdown
- Name (Affiliation)
```

## Adding Agenda Information

In the relevant event folder, edit `agenda.md` with the daily schedule:

```markdown
## Day 1: Date

| Time | Session | Speaker/Lead |
|------|---------|-------------|
| 09:00 | Welcome | Organiser |
| 09:30 | Talk Title | Speaker Name |
```

## General Guidelines

- Keep entries factual and concise
- Include links where available (slides, videos, code repos)
- Use the existing format in each file as a guide
- If you are unsure about something, open an issue first
- Be respectful of all community members in descriptions and attributions

## Questions?

Open an issue or contact the .Astronomy Foundation.
